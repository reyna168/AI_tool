# LabelImg 與 YOLO 標註重點

## 1. LabelImg 是什麼

LabelImg 是一個影像標註工具，用來替圖片中的物件建立 Bounding Box（外框），產生物件偵測模型所需的標註資料。

主要用途：

* Object Detection（物件偵測）
* YOLO
* Pascal VOC
* TensorFlow Object Detection




# 4. YOLO 標註格式

YOLO 每一列代表一個物件：

```
class x_center y_center width height
```

例如：

```
0 0.512 0.438 0.210 0.375
```

表示：

* class = 0
* 中心 x = 0.512
* 中心 y = 0.438
* 寬 = 0.210
* 高 = 0.375

注意：

所有座標都是 **正規化（Normalized）**，範圍介於 **0～1**。

---

# 5. YOLO 座標公式

假設圖片：

```
640 × 480
```

Bounding Box：

```
xmin = 100
ymin = 50
xmax = 300
ymax = 250
```

計算：

```
x_center = (100 + 300) / 2 = 200
y_center = (50 + 250) / 2 = 150

width = 300 - 100 = 200
height = 250 - 50 = 200
```

正規化：

```
x = 200 / 640 = 0.3125
y = 150 / 480 = 0.3125
w = 200 / 640 = 0.3125
h = 200 / 480 = 0.4167
```

YOLO：

```
0 0.3125 0.3125 0.3125 0.4167
```

# 6. classes.txt

類別依序編號：

```
person
car
cat
dog
helmet
```

YOLO 對應：

```
0 person
1 car
2 cat
3 dog
4 helmet
```

LabelImg 會依類別順序使用索引，因此類別順序應保持一致。

# 7. 資料集結構（YOLO）

```
dataset/
│
├── images/
│   ├── train/
│   ├── val/
│   └── test/
│
├── labels/
│   ├── train/
│   ├── val/
│   └── test/
│
└── data.yaml
```

其中：

* `images/`：存放圖片
* `labels/`：存放對應的 YOLO `.txt`
* `data.yaml`：定義資料集路徑與類別

---

# 8. data.yaml 範例

```yaml
train: images/train
val: images/val

nc: 3

names:
  0: person
  1: helmet
  2: vest
```

# 10. 標註注意事項

* Bounding Box 應盡量貼合物件邊界，避免過大或過小。
* 同一類別需使用一致的名稱（例如不要混用 `Car`、`car`、`CAR`）。
* 每張圖片都應有對應的 `.txt` 標註檔；若圖片沒有任何物件，通常保留一個空白的 `.txt`。
* 訓練集、驗證集與測試集應使用相同的類別編號與 `data.yaml` 設定。
* 標註品質直接影響模型訓練效果，因此應檢查標註是否正確且一致。
